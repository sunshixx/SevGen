运维工程师的回复模板

## 故障处理相关回复

### 故障确认回复
```
收到故障报告，我立即开始处理：

**故障概况**
- 故障时间：[具体时间]
- 影响范围：[受影响的系统/用户]
- 严重程度：[P0/P1/P2级别]
- 当前状态：[正在调查/已定位/修复中]

**处理进展**
1. 已确认故障现象和影响范围
2. 正在收集相关日志和监控数据
3. 预计[时间]内提供进一步更新

我会每[15/30]分钟更新一次处理进展，有任何问题请随时联系我。
```

### 故障进展更新
```
故障处理进展更新 - [时间]

**最新进展**
- 根因分析：[已定位的问题原因]
- 修复方案：[采用的修复策略]
- 执行进度：[当前执行状态]
- 预计恢复时间：[预估的恢复时间]

**已完成的工作**
1. [具体完成的步骤1]
2. [具体完成的步骤2]
3. [具体完成的步骤3]

**下一步计划**
1. [下一步要执行的操作1]
2. [下一步要执行的操作2]

继续密切监控系统状态，下次更新时间：[具体时间]
```

### 故障恢复通知
```
故障已恢复 - [时间]

**恢复确认**
- 系统功能：✅ 已完全恢复
- 性能指标：✅ 已回到正常水平
- 数据完整性：✅ 已验证无损失
- 用户访问：✅ 已恢复正常

**故障总结**
- 故障持续时间：[X小时X分钟]
- 根本原因：[简要说明根因]
- 修复措施：[采取的修复措施]
- 影响评估：[对业务的影响评估]

**后续行动**
1. 将在[时间]组织故障复盘会议
2. 制定预防措施防止类似问题再次发生
3. 更新监控规则和应急预案

感谢大家在故障处理过程中的配合和支持。
```

```

**维护信息**

- 维护系统：[具体系统名称]
- 维护类型：[升级/补丁/配置变更]
- 影响范围：[受影响的功能/用户]

**维护内容**
1. [具体维护项目1]
2. [具体维护项目2]
3. [具体维护项目3]

**风险评估**
- 风险等级：[低/中/高]
- 回滚方案：已准备完整的回滚预案
```
